# Changesets

Hello and welcome! This folder has been automatically generated by `@changesets/cli`,
a build tool that works with multi-package repos, or single-package repos to help you
version and publish your code. You can find the full documentation for it
[in our repository](https://github.com/changesets/changesets)

## What is this for

Every PR that changes the public surface of `@velxio/sdk` (any file under
`packages/sdk/src/`) **must include a changeset**. A changeset is a small
markdown file in this folder that:

1. Lists the packages that change.
2. Picks a SemVer bump (`patch` / `minor` / `major`).
3. Has a one-line summary that ends up in the CHANGELOG.

## How to create one

From the repo root:

```bash
npx changeset
```

This walks you through:
- which package(s) changed (only `@velxio/sdk` for now)
- the bump type
- a short description

It writes a file like `.changeset/orange-dragons-dance.md`. Commit that file
with your PR.

## What happens at release time

On every push to `master`, the **Release** GitHub Action:

1. Looks at the pending changeset files.
2. Opens (or updates) a "Version Packages" PR that bumps the package
   versions and updates `CHANGELOG.md`.
3. When that PR is merged, the same workflow runs `changeset publish`
   — which publishes the new version to npm.

You should never edit `CHANGELOG.md` or bump version numbers by hand —
`changeset version` does that.
